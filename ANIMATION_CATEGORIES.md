# 金融记账应用动画特效分类指南

## 🎨 动画分类体系

本应用提供了 **14种核心动画**，按照使用场景分为 **6大分类**，每个分类包含 **12个扩展动画建议**。

## 💡 设计原则

### 🎨 一致性原则
- 同类动画使用统一的时长和缓动曲线
- 相似的交互保持一致的视觉反馈

### ⚡ 性能原则
- 避免过度动画影响性能
- 使用高效的动画实现方式
- 合理控制动画时长 (200-1000ms)

### 🎯 用户体验原则
- 动画应增强而非干扰用户操作
- 提供适当的视觉层次和引导
- 考虑无障碍访问需求

## 🔧 技术实现建议

### 📱 动画框架选择
- **简单动画**: `AnimatedBuilder` + `Tween`
- **复杂动画**: `AnimationController` + `CurvedAnimation`
- **组合动画**: `TweenSequence` (谨慎使用)

### 🎨 缓动曲线推荐
- **按钮点击**: `Curves.easeInOut` (200ms)
- **列表操作**: `Curves.easeInOutCubic` (300ms)
- **成功反馈**: `Curves.elasticOut` (800ms)
- **加载状态**: `Curves.easeInOut` (1500ms)

### 📏 动画参数标准化
```dart
// 标准动画时长
const Duration kShortDuration = Duration(milliseconds: 200);
const Duration kMediumDuration = Duration(milliseconds: 400);
const Duration kLongDuration = Duration(milliseconds: 800);

// 标准缓动曲线
const Curve kPrimaryCurve = Curves.easeInOut;
const Curve kSuccessCurve = Curves.elasticOut;
const Curve kLoadingCurve = Curves.easeInOut;
```

这个分类体系涵盖了金融记账应用的所有主要交互场景，每个分类都提供了12个动画建议，确保功能的完整性和一致性。
