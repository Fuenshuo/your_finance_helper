# 奖金添加功能优化方案（基于真实用户反馈）

## 🎯 核心原则

**"用户不是在配置奖金规则，而是在记录一笔已发生的收入。"**

---

## 💥 用户差评总结

| 当前设计问题 | 用户真实感受 | 影响 |
|------------|------------|------|
| 授予日期 vs 归属日期 | "这 App 当我是 CFO？" | ⭐⭐⭐⭐⭐ 严重 |
| 字段过多（7个字段） | "我只是记个账，不是填 IRS 表格" | ⭐⭐⭐⭐⭐ 严重 |
| 季度奖金月份网格 | "我手机屏幕就这么大，你让我点 12 个格子？" | ⭐⭐⭐⭐ 高 |
| 对话框过长需滚动 | "保存按钮在哪？" | ⭐⭐⭐⭐ 高 |
| "年度总额"概念 | "我哪知道年度总额？我只知道每次发 8000" | ⭐⭐⭐ 中 |

---

## ✅ 最终优化方案

### 方案架构：快速模板 + 极简自定义

```
┌─────────────────────────────────────┐

## 📱 方案一：快速模板（90% 场景）

### 1.1 年终奖快速添加

**界面设计：**
```
┌─────────────────────────────────────┐
│  添加年终奖                          │
├─────────────────────────────────────┤
│                                      │
│  金额                                │
│  ┌──────────────────────────────┐  │
│  │ ¥ 150,000                    │  │
│  └──────────────────────────────┘  │
│                                      │
│  发放日期                            │
│  ┌──────────────────────────────┐  │
│  │ 2025年12月31日        [选择] │  │
│  └──────────────────────────────┘  │
│                                      │
│  💡 提示：年终奖单独计税，与工资分开 │
│                                      │
│  [ 取消 ]        [ 添加到收入 ]     │
└─────────────────────────────────────┘
```

**字段说明：**
- ✅ **金额**：必填，数字输入框
- ✅ **发放日期**：必填，默认当年 12 月 31 日
- ❌ **移除**：授予日期、归属日期、发放月份、年度总额等所有专业字段

**自动填充：**
- 名称：自动生成 "2025 年终奖"（根据年份）
- 类型：自动设置为 `yearEndBonus`
- 发放日期：默认当年 12 月 31 日

**用户操作流程：**
1. 点击"年终奖"按钮
2. 输入金额（或确认自动填充的金额）
3. 确认发放日期（或使用默认值）
4. 点击"添加到收入"
5. **完成！**（3步，10秒内完成）

│  │ 2025年3月31日         [选择] │  │
│  └──────────────────────────────┘  │
│                                      │
│  [ 取消 ]        [ 添加到收入 ]     │
└─────────────────────────────────────┘
```

**字段说明：**
- ✅ **单次金额**：必填，用户输入每次发放的金额（不是年度总额！）
- ✅ **发放月份**：下拉选择，预设选项：
  - 每季度末（3/6/9/12月）← 默认
  - 每季度初（1/4/7/10月）
  - 自定义（展开月份选择）
- ✅ **首次发放日期**：必填，默认当年 3 月 31 日

**自动计算：**
- 年度总额 = 单次金额 × 4（如果选择每季度末）
- 发放次数 = 4（自动计算）
- 名称：自动生成 "2025 季度奖"

**用户操作流程：**
1. 点击"季度奖"按钮
2. 输入单次金额（如：8000）
3. 选择发放月份（或使用默认"每季度末"）
4. 确认首次发放日期（或使用默认值）
5. 点击"添加到收入"
6. **完成！**（4步，20秒内完成）

## 🎨 UI/UX 细节优化

### 3.1 术语优化

| 当前术语 | 优化后 | 原因 |
|---------|--------|------|
| 授予日期 | 发放日期 | 用户只关心"什么时候到账" |
| 归属日期 | （合并到发放日期） | 99% 用户不需要区分 |
| 奖金年度总额 | 单次金额 | 用户只知道每次发多少 |
| 发放次数 | （移到高级设置） | 系统自动计算 |

### 3.2 交互优化

**月份选择优化：**
- ❌ 当前：12 个月份网格（占用大量空间）
- ✅ 优化后：下拉选择预设选项
  - 每季度末（3/6/9/12月）
  - 每季度初（1/4/7/10月）
  - 自定义（展开月份选择器）

**日期选择优化：**
- ❌ 当前：弹窗日期选择器（打断流程）
- ✅ 优化后：内联日期选择器（iOS 风格）

**金额输入优化：**
- ✅ 自动格式化：输入 "150000" → 显示 "150,000"
- ✅ 自动填充：十三薪自动填充基本工资
- ✅ 历史记录：点击"历史"按钮快速复用

### 3.3 移动端优化

**对话框高度：**
- ✅ 最大高度：屏幕的 80%（避免滚动）
- ✅ 固定底部按钮：保存/取消按钮始终可见
- ✅ 键盘适配：输入框聚焦时自动滚动到可视区域

**单手操作优化：**
- ✅ 主要操作按钮在底部右侧（右手拇指区域）
- ✅ 取消按钮在底部左侧（避免误触）
- ✅ 输入框足够大，易于点击

## 💡 关键洞察

### 用户真实需求

1. **记录事实，而非配置规则**
   - 用户只关心："我收到了多少钱，什么时候收到的"
   - 不需要知道："这笔钱的会计归属期是什么时候"

2. **快速完成，而非完美配置**
   - 90% 用户场景：年终奖、十三薪、季度奖
   - 10% 用户场景：其他复杂奖金（可以走自定义路径）

3. **移动端优先，而非桌面端思维**
   - 避免长表单滚动
   - 避免复杂交互（如月份网格）
   - 避免专业术语

### 设计原则

1. **默认简单，按需复杂**
   - 快速模板：极简字段（2-4 个）
   - 自定义表单：简化字段（5 个）
   - 高级设置：折叠隐藏（按需展开）

2. **用户语言，而非系统语言**
   - "发放日期"而非"授予日期"
   - "单次金额"而非"年度总额"
   - "每季度末"而非"3/6/9/12月"

3. **智能默认，减少输入**
   - 自动填充基本工资（十三薪）
   - 自动生成名称（根据类型和年份）
   - 自动计算年度总额（季度奖）

   - 避免长表单滚动
   - 避免复杂交互（月份网格）
